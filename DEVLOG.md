## 2026-07 第2週（07-11〜07-12）

**AWS 構成の決定・ローカルサーバ稼働・決定的ゲートの大幅拡張・RRF の管轄確定。**

- **AWS 維持費の総ざらい → Data API 型に構成決定**: デプロイ前に「一切使わず放置した場合の月額」を公式料金表（Bulk Pricing API）から全構成分算出。旧構成案には罠が 2 つ埋まっていた——①RDS Proxy は auto-pause と非互換（公式明記）な上に Serverless v2 へは最低 8 ACU 分の時間課金（Proxy だけで月 $146）②Interface 型 VPC エンドポイントは稼働状態と無関係に毎時課金。放置時 $232/月 → **$0.84/月** の Data API 型（Lambda を VPC 外に置き、HTTPS + IAM で Aurora に入る）へ再設計した。〔記事化候補: "RDS Proxy と auto-pause は両立しない"〕
- **ローカル API サーバ + Web UI**: FastAPI + 静的 1 ファイル UI（ビルド工程ゼロ＝S3 配信前提の構成をローカルで検証）。検索経路バッジとレイテンシを UI に表示。systemd user unit で VM 起動時に自動起動・死活監視（kill 後 5 秒で自動復旧を実測）。
- **ローカル 7B をルーターとして実戦配備**: `ROUTER_BACKEND=ollama` で API サーバのルーターをローカル 7B（$0）に切替可能に。検証層（後述）はプロバイダ非依存なので同じ保護が掛かる。
- **DB アクセス層の切替設計**: SQL 実行 14 箇所を単一ドライバ層（psycopg2 ⇔ Aurora Data API）に集約。移行が検索品質に完全透過であることを eval 全指標一致で証明。
- **決定的ゲートを 5 系統へ拡張**（すべて「本線 eval の全指標一致」を回帰条件に導入）: 生得キーワード 33 語の肯定・否定（「速攻を持たないクリーチャー」——embedding は否定文を原理的に理解しない）／日本語カードタイプ語／P/T 列間関係（「パワーとタフネスが同じ」）／部族 65 種（「蟹」2/10 → 5/5）／カード名部分一致（「カード名にナヒリとつく」1/10 → 10/10）。〔記事化候補: "embedding は否定文を理解しない"〕
- **LLM 出力の検証層を 3 段に**: 数値幻覚ガード（既存）に加え、排他境界の ±1 補正（LLM は「9より小さい→max=8」に成功しつつ同一クエリの「7より大きい→min=7」で失敗する、と実測）と type_filter 幻出ガードを追加。〔記事化候補: "LLM の出力を信用しない検証層の設計"〕
- **RRF の管轄を腕別アブレーションで確定**: 「RRF という仕組み自体が欠陥では」という骨格への疑いを、判定基準を事前固定した 6 条件の実測で検証。結果——FTS 2 腕は曖昧な意味の層で +0.088 の相補寄与（英=キーワード拡張の受け皿・日=日本語定型句の受け皿・寄与はほぼ加法的）・均等重みは局所最適・答えが一意に決まる層に合議を使うと人気者バイアス（実測）。「欠陥」ではなく「管轄が一意に決まる層から正しく剥奪され、残った曖昧な意味の層で稼ぐ現役」と結論。
- **クエリ観測ログ（query_log）稼働**: 全クエリの経路・ルーター解析・レイテンシを記録。直行路 11ms vs ルーター経由 15,024ms のような実測が SQL 一発で取れる。辞書拡張の候補抽出（human-in-the-loop 語彙学習）の観測基盤。
- 週間の学び: **UI で遊ぶ → query_log が証拠を取る → 検証層/ゲートで塞ぐ**というループが確立した（この週のバグ・穴は全部、作者自身が UI で遊んでいて発見した）。

## 記事化候補（ブログ執筆キュー）

1. RDS Proxy と auto-pause は両立しない——Aurora Serverless v2 の放置時維持費を公式料金表から総ざらいした
2. embedding は否定文を理解しない——「速攻を持たないクリーチャー」を SQL の NOT で解く
3. LLM の出力を信用しない——数値幻覚・境界演算・型幻出の 3 段検証層
4. 個人開発の RAG に評価基盤を作る——NDCG 0.574 → 0.811 の系譜と「未ラベル混入率 0%」の方法論
5. RRF は欠陥か？——腕別アブレーションで「合議の管轄」を実測した
6. ローカル開発に Nginx が要らなかった理由——サーバの階層と玄関番の仕事
